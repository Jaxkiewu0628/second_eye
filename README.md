## TL;DR(当前状态,诚实版)
做**端侧优先**的可穿戴 AI 眼镜。**技术可行性已大量验证;但产品方向正在重定** ——
原定的"高尔夫码数"切入点被真实用户反馈否决,目前在用"**从端侧强项反向搜蓝海痛点**"
的方式重新找切入点(不跟风、找真痛点)。

技术不是主要风险,**go-to-market 方向才是当前最大未决项。**

---

## 核心资产 / 论点
- **端侧优先,云端增强**:核心功能离线全可用,云端只做增强、永不依赖
  (对手如 Ray-Ban Meta 断网就废,我们不)。这是差异化主线。详见 `docs/rk3576_fit_and_voice_workflow.md`。
- **诚实的反方**:在"户外人人已有 Garmin"的市场,standalone 端侧可能是负担而非优势
  (ENGO 那种"给现有手表配轻 HUD"的伴侣式路线更易被接受)。此路线之争未定论。

---

## ✅ 已验证(技术,基于真实硬件公开实测,非外推)
- **STT** Paraformer RTF 0.29(快 3.4×)/ **TTS** Matcha+Vocos RTF 0.13,首音 ~320ms(RK3576 实测)
- **YOLOv8n** RK3576 真机 ~30fps;工具链(ONNX→RKNN)跑通,量化需 x86
- 避开了旧板 **RV1106 陷阱**(STT 曾 57× 慢于实时)
- **占空比架构**(唤醒协处理器 + 主控常睡)可行,是全行业做法
- **外观/结构**:刀锋镜腿方案 + 3D 模型 + 给设计团队的 ID 规格书

## ❌ 已否决 / 已学到(同样重要)
- **高尔夫楔子**:真实球友否决——"属于作弊"(赌球/竞技文化)、球童已代测、Garmin 表已戴、专注打球用处不大
- **"一副眼镜解决所有运动"**:万能平台陷阱,V1 不能这么做
- **RK3576 ≠ 量产芯片**:它是边缘盒/工业平板芯片(功耗 1.2-2.8W,不合 <50g 镜框),**仅作验证平台 / 分体盒**
- **Snapdragon AR1 买不到**(NDA),且用它=站巨头同款平台被商品化

## 🔴 待决策 / 进行中
- **切入点 / 垂直**:重找(当前:端侧优点 → 蓝海痛点,候选含无障碍/工业/应急/户外)
- **显示光机 + 波导**:铁三角真正瓶颈,**未选型**(MicroLED 轻省但难拿,微 OLED+Birdbath 是 startup 现实路径)
- **量产主控**:候选 Unisoc W517 / Rockchip RV1126B(同 RKNN 生态,验证可迁移)/ 爱芯 AX630C —— 需验 NPU 实际能力 + 镜框散热
- **standalone vs 手表伴侣 HUD** 路线之争
- **需求真实承诺**:用户付费/定金尚未验证(只到"愿付"口头阶段)

## 工作方式(给接手的人)
本项目按**硬件开发**组织(非软件仓库流程):
- 先验证后成文,少产出文档;结论**先查证不揣测**(引用来源,实测 vs 估算分清)
- 做衡量时**多角度、不一味说好**(列代价、反方、什么情况会反转),重大判断二次验证
- 详见 `CLAUDE.md`

# Second Eye — 视障辅助 AI 眼镜

> 单一事实源是 [`STATUS.md`](STATUS.md)，先读它。本文是给新人的鸟瞰。
> 最后更新：2026-06-29

---

## TL;DR

做帮助视障人士独立出行的 AI 眼镜。
**摄像头 + 端侧 YOLOv8 识别障碍物 → 骨传导语音告知方位**（"前方 1.5 米有台阶"）。
核心功能完全离线，断网不影响使用——这对视障用户是安全要求，不是特性。

本项目脱胎于 **YCane**（2023 Conrad Challenge 参赛项目，实用新型专利已申请），
用 RK3576 端侧 AI 技术栈将原本 ESP32 + ToF 的简单方案全面升级。

---

## 痛点（已验证，非假设）

- 全球 1.73 亿视障人口，53% 出行需要帮助
- 现有方案：传统手杖（不识别头部障碍）、OrCam $3500（读文字为主）、WeWalk $500（基础 ToF，无 AI）
- **没有一个方案能做到：离线 + 语义理解 + 轻量眼镜形态**

---

## 技术路线

```
摄像头 ──→ NPU（YOLOv8）──→ "台阶/行人/车辆/门"
ToF LiDAR ───────────────→ 精确距离（cm 级）
         融合 ──→ 骨传导："前方 1.5 米有台阶，右侧 2 米有行人"
麦克风 ──→ 本地 STT ──→ 用户指令 ──→ 本地推理 / 云端增强
```

**不需要显示屏**——去掉了之前最大的硬件瓶颈。

---

## 已验证（基于 RK3576 公开实测）

| 能力 | 结果 |
|------|------|
| STT（Paraformer） | RTF 0.29，实时 3.4× |
| TTS（Matcha+Vocos） | RTF 0.13，首音 ~320ms |
| YOLOv8n 推理 | ~30fps，RKNN 工具链跑通 |
| H.265 视频管道 | 1080p 实测，产品路径预估 <100ms |

---

## 已否决

- ❌ 高尔夫/运动切入：被真实用户否决
- ❌ AR 显示屏：视障用户不需要，去掉
- ❌ RK3576 直接量产：功耗 1.2-2.8W，镜框装不下，仅作验证平台

---

## 待决策

- **ToF LiDAR 选型**（VL53L5CX vs OPT8241）
- **YOLO 微调**：通用 COCO vs 视障专用数据集（台阶/盲道/斑马线）
- **量产主控**：RV1126B（3 TOPS，同 RKNN 生态）或 AX630C
- **法规路径**：是否走医疗器械认证

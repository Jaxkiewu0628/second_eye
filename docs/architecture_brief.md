# AR 运动眼镜硬件架构简报

> 功耗分析 · 算力需求 · 唤醒词方案 · 一体式可行性论证  
> 2026-06-26

---

## 一、我们要解决什么问题

上一版简报确定了主芯片用 **RK3576（6 TOPS NPU）**，但留下了三个悬而未决的问题：

1. **唤醒词怎么实现？** 低功耗芯片怎么选，VAD 在哪里跑？
2. **这堆功能到底需要多少算力（TOPS）？** 1080p 视频 + 视觉模型 + 音乐播放 + 两秒内响应
3. **电池够不够？** 一体式眼镜能跑 4-5 小时吗，还是必须做分体结构？

本文给出完整答案。

---

## 二、核心结论（先看这里）

| 问题 | 结论 |
|------|------|
| 需要多少 TOPS | **2.5-4 TOPS**（YOLOv8n + STT + TTS 同时跑，含 5× headroom） |
| 唤醒词跑在哪里 | 专用低功耗 MCU，主 SoC 完全睡眠 |
| 推荐唤醒词芯片 | **Syntiant NDP120**（8-12mW，¥60-90） |
| 一体式是否可行 | ✅ **可行**，占空比模型下 300mAh 可跑 14+ 小时 |
| 分体结构何时需要 | 仅当需要**全程持续录像**（非主用例） |

## 三、算力需求拆解

### 关键认知：1080p 录像不消耗 NPU

H.265 视频编码由 RK3576 硬件编码器（MPP）完成，**不占用 NPU 或 CPU**。视觉模型的输入分辨率也不需要 1080p——通常缩放至 640×480 推理即可满足两秒内响应。

| 功能 | 实现方式 | 所需算力 |
|------|---------|---------|
| 1080p @ 60fps 录制 | 硬件编码（MPP） | **0 TOPS**（专用电路） |
| YOLOv8n 目标检测，10fps | NPU | 0.4-0.6 TOPS |
| 放大区域二次检测（局部 filter） | NPU，输入 320×240 | 0.1-0.2 TOPS |
| STT 本地语音识别 | NPU | 0.3-0.5 TOPS |
| TTS 本地语音合成 | NPU | 0.1-0.2 TOPS |
| **所有功能同时运行** | | **~0.9-1.5 TOPS** |
| **× 5 倍 headroom（行业标准）** | | **需要 4.5-7.5 TOPS** |

**RK3576 提供 6 TOPS，恰好覆盖全功能同时运行的 5× headroom。**

> **备注**：视觉模型 10fps 完全足以实现两秒内响应——用户发出指令后，10fps 意味着每 100ms 更新一帧检测结果，链路延迟由网络（云端 AI）主导，不是 NPU。

## 四、唤醒词模块

### 唤醒词应该跑在哪里？

| 方案 | 待机功耗 | 体验 | 结论 |
|------|---------|------|------|
| **专用低功耗 MCU**（独立芯片） | **0.5-20mW** | 即时响应，50-150ms | ✅ 推荐 |
| 主 SoC 低功耗模式（RK3576 内置 Hifi4 DSP） | 50-100mW | 可用，但费电 | ✅ V1 零成本方案 |
| 手机 App 监听 | 眼镜~0，手机消耗高 | 延迟 500ms-2s | ❌ 体验差 |
| 云端持续推流 | WiFi 常开 500-1500mW | 延迟 1-3s，耗电 | ❌ 不适合 |

### 专用唤醒词芯片对比

| 芯片 | 待机功耗 | 监听功耗 | 唤醒延迟 | 准确率 | 国内价格 | 可买到 |
|------|---------|---------|---------|--------|---------|--------|
| **Syntiant NDP120** | 0.5mW | **8-12mW** | **50ms** | 95-99% | ¥60-90 | ✅ |
| ESP32-S3（原型首选） | 10µW | 80-150mW | 200-400ms | 85-92% | ¥20-50 | ✅ |
| NationalChip GX8002 | 2-3mW | 15-20mW | 150-300ms | 90-95% | 较贵 | ✅（闪极在用） |
| Ambiq Apollo4 | 5µW | 20-50mW | 200-500ms | 93-97% | ¥80-130 | ⚠️ |

**推荐路线：**
- **V1 验证**：直接用 RK3576 内置 Hifi4 DSP + 按键触发，零硬件成本，专注跑通 YOLO-STT-TTS 链路
- **V2 量产**：加 Syntiant NDP120（¥60），8-12mW，达到 Snapdragon AR1 同等唤醒体验

## 五、一体式可行性：占空比架构

### 传统思路的误区

按"全部功能同时持续运行"计算功耗，必然得出"需要 650-850mAh 电池 → 镜腿装不下 → 必须分体"的结论。

但 AR 运动眼镜的真实使用场景不是这样的。

### 实际使用模式（以高尔夫为例）

```
一场球（5 小时）：
  - 走路、等待：占 ~95% 时间，不需要 AI 介入
  - 每次击球前问一次距离/轨迹：10-15 秒激活
  - 共约 60-80 次击球 → 激活时间合计约 15 分钟
```

### 占空比架构下的功耗

```
系统架构：
  ┌──────────────────────────────────────────────────┐
  │  NDP120（唤醒词监听，常亮）         ~10mW        │
  │  RK3576（深度睡眠，suspend-to-RAM） ~15-20mW     │
  │  WiFi / 摄像头（断电）             ~0mW          │
  │  合计待机功耗：                     ~25-30mW     │
  └──────────────────────────────────────────────────┘
          │ 检测到唤醒词（GPIO 中断）
          ▼  唤醒延迟 200-500ms
  ┌──────────────────────────────────────────────────┐
  │  RK3576 全速（NPU + CPU + WiFi + 摄像头）        │
  │  活跃功耗：                         ~600mW       │
  │  单次持续时长：                      3-10 秒     │
  └──────────────────────────────────────────────────┘
          │ 任务完成
          ▼
  回到深度睡眠
```

### 电池计算

| 时段 | 功耗 | 时长 | 耗电 |
|------|------|------|------|
| 深度睡眠 | 25mW | 4h 45min | 0.119 Wh |
| 激活处理 | 600mW | 15min | 0.150 Wh |
| **合计** | | **5 小时** | **0.27 Wh** |

```
所需电池容量 = 0.27Wh ÷ 3.7V ≈ 73mAh（理论最低）

实际配置 300mAh（三节分布，闪极标准）：
  可用能量 = 1.11Wh
  续航余量 = 1.11 ÷ 0.27 ≈ 4×
  等效续航 ≈ 20 小时
```

**一体式方案电池完全够用，300mAh 在高尔夫场景下有 4 倍余量。**

### 这正是 Snapdragon AR1 的设计哲学

AR1 的 Hexagon DSP 在 10-20mW 下持续监听唤醒词，主 CPU 完全关闭——Ray-Ban Meta 49g、210mAh 电池能待机一整天，根本原因不是 AR1 "很省电"，而是它**大部分时间根本不工作**。

我们用 NDP120 + RK3576 suspend 的组合，可以复制同等效果。

## 六、功能模式矩阵

| 使用场景 | 功耗 | 300mAh 续航 | 是否需要分体 |
|---------|------|------------|------------|
| 高尔夫辅助（触发式 AI） | ~50-80mW 均值 | **14-22 小时** | ❌ 不需要 |
| 运动追踪（每分钟触发） | ~80mW 均值 | ~14 小时 | ❌ 不需要 |
| 语音对话（偶发） | ~60mW 均值 | ~18 小时 | ❌ 不需要 |
| **全程持续录像（连续）** | ~600mW 持续 | **~1.8 小时** | ⚠️ 需分体或大电池 |

> 结论：**全程持续录像是唯一让一体式续航不够的场景**，而这不是 AR 运动辅助眼镜的核心功能。按需录制（触发式录像）完全可行。

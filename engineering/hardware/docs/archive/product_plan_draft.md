# AR 运动眼镜产品计划草案

> 双模架构 · 产品分层 · 技术路线  
> 2026-06-26 · 草案 v0.1

---

## 一、产品定位

**比非运动眼镜（Ray-Ban Meta）更轻便、便宜；比运动眼镜（GPS 追踪器）能力更突出、先进。**

核心差异化：
- 全自主式（Standalone）— 无需手机，眼镜本体完成所有 AI 推理
- 运动场景优先 — 高尔夫轨迹、目标追踪、实时语音辅助
- 可选扩展 — 需要长续航/持续录像时，外接计算盒而非换产品

---

## 二、双模架构概念

### 核心思路

**一副眼镜，两种能力上限。** 软件层自动检测计算盒是否在线，透明切换处理模式，用户和 App 无感知。

```
┌─────────────────────────────────────────────────────┐
│                    眼镜本体                          │
│  摄像头 · 麦克风 · IMU · 扬声器 · 显示模块          │
│  主芯片：RK3576（6 TOPS NPU）                       │
│  电池：300mAh（三节分布镜腿）                        │
└────────────┬────────────────────────────────────────┘
             │
     有计算盒？
    ┌─────────┴─────────┐
    │ 否                │ 是
    ▼                   ▼
独立模式            瘦客户端模式
（眼镜作处理中心）   （眼镜作收发器）
```

## 三、模式详解

### 模式一：独立模式（Standalone）

眼镜自主完成全部 AI 推理，无需任何外部设备。

**适用场景：** 日常运动辅助、触发式语音 AI、目标检测

**工作方式：**
```
唤醒词检测（NDP120，常亮 10mW）
    │ 检测到唤醒词
    ▼
RK3576 从深度睡眠唤醒（~300-500ms）
    │
    ├── NPU：YOLOv8n 目标检测
    ├── NPU：本地 STT 语音识别
    ├── WiFi：Claude API 云端 AI
    └── NPU：本地 TTS 语音合成
    │
    ▼
播放回答 → 回到深度睡眠
```

**续航估算（高尔夫场景，5小时）：**

| 状态 | 功耗 | 时长 | 耗电 |
|------|------|------|------|
| 深度睡眠 + 唤醒词监听 | ~25mW | 4h 45min | 0.119 Wh |
| 激活处理（~60次 × 10秒） | ~600mW | 15min | 0.150 Wh |
| **合计** | | **5小时** | **0.27 Wh** |

300mAh 电池可用能量 1.11 Wh，**有 4 倍余量，续航约 20 小时。**

---

### 模式二：瘦客户端模式（Tethered）

眼镜仅负责采集和显示，所有 AI 推理卸载到计算盒。

**适用场景：** 持续录像、高频视觉处理、极限运动全程记录

**工作方式：**
```
眼镜：
  摄像头 → H.265 硬件编码 → WiFi 6 推流 → 计算盒
  计算盒 → 结果数据（AR叠加/语音）→ WiFi → 眼镜显示/播放

计算盒：
  接收视频流 → NPU 推理 → 返回结果
  本地大容量存储录像
  大电池持续供电（含给眼镜反向充电）
```

**眼镜在瘦客户端模式下的功耗：**

| 模块 | 独立模式 | 瘦客户端模式 |
|------|---------|------------|
| NPU 推理 | 150-300mW | **0mW** |
| CPU | 200-300mW | **50-80mW** |
| 摄像头 + 硬件编码 | 80-120mW | 80-120mW |
| WiFi 收发 | 150-200mW | 150-200mW |
| **合计** | **~600mW** | **~280-400mW** |

300mAh 电池在持续全功能模式下：**2.8-4 小时**（对比独立模式持续约 1.8 小时，提升 70%）。

## 四、关键技术：软件层透明切换

这是区别于 Rokid + Station / XREAL + Beam Pro 的核心差异。

### 竞品的问题

| 产品 | 问题 |
|------|------|
| Rokid Max + Station 2 | 离开 Station 功能残缺，基本只剩显示 |
| XREAL Air 2 + Beam Pro | 插拔切换，用户有感知，软件不统一 |
| Magic Leap 1 | 必须有线连接，无独立模式 |

### 我们的方案

```
App / SDK 层（统一接口）
        │
        ▼
  InferenceRouter（推理路由器）
        │
   ┌────┴────┐
   │         │
本地引擎    远程引擎
（RK3576    （计算盒
  NPU）      RK3576）
```

- 计算盒上线 → 路由器自动将推理任务发给计算盒
- 计算盒离线 → 路由器自动回退到本地 NPU
- 切换过程 <100ms，App 层无需改代码
- **同一副眼镜，软件一套，两种硬件形态**

## 五、计算盒规格建议

计算盒与眼镜用同款 SoC，软件完全复用：

| 配置 | 规格 | 说明 |
|------|------|------|
| 主芯片 | **RK3576**（同眼镜） | 软件 100% 复用，无额外开发成本 |
| 电池 | 3000-5000mAh | 持续全功能 5-8 小时 |
| 反向充电 | 支持给眼镜充电 | 计算盒即充电宝 |
| 存储 | 128-256GB | 本地录像存储 |
| 散热 | 被动散热 + 铝壳 | 无风扇，运动场景无噪音 |
| 形态 | 口袋 / 腰包 / 背包扣 | ~150g |
| 连接 | WiFi 6（主）+ BT 5.3（控制） | 视频流走 WiFi，指令走 BT |

## 六、产品 SKU 与定价策略

```
入门款（眼镜单体）           专业款（眼镜 + 计算盒）
─────────────────           ──────────────────────
目标价：¥699-799             目标价：¥1199-1499
续航：4-5h（触发式）          续航：8h+（持续全功能）
用例：运动辅助 / 日常          用例：极限运动 / 专业记录
                             
用户路径：先买入门款 → 需要更多续航/算力 → 单独购买计算盒（¥499-699）
```

**计算盒可以单独销售，不绑定眼镜版本。** 眼镜硬件迭代，计算盒不换；计算盒升级（更强 SoC），眼镜不换。

## 八、开发路线

### V1：链路验证（当前阶段）

**目标：** 跑通 唤醒 → STT → AI → TTS 完整链路

- 硬件：野火鲁班猫3（RK3576）+ IMX415 摄像头 + USB 麦克风
- 软件：Sherpa-ONNX STT + Claude API + Piper TTS
- 验证点：端到端延迟 <2s，YOLO 推理 >10fps
- **不做唤醒词芯片，用按键模拟触发**

### V2：工程样机（获投前）

**目标：** 眼镜形态 + 双模切换原型

- 硬件：RK3576 SoM 自设计底板，300mAh 三节电池
- 加入 Syntiant NDP120（真实唤醒词）
- 实现 InferenceRouter 基础版（本地/远程自动切换）
- 计算盒：树莓派或鲁班猫3 跑服务端，验证协议

### V3：量产版（获投后）

**目标：** 量产就绪

- 评估 MediaTek 下代 AR 芯片（Meta 路线，2026-27）
- 计算盒量产设计（RK3576 + 定制外壳）
- 显示模块选型（光波导 vs Micro-OLED）
